Sanftanlauf von Extrudern mit Frequenzumrichter
Linux Modbus TCP Steuerung und Getriebeschutz

Autor: Igor Krjazhev, Verfahrenstechnologe & Kunststoffberater

1. Warum der Direktstart den Extruder zerstört

Der Direktstart eines Asynchronmotors erzeugt einen Einschaltstrom, der um das 5- bis 7-fache über dem Nennstrom liegt. Bei einem 55-kW-Motor (Nennstrom 100 A) bedeutet das 500–700 A Anlaufstrom. Diese Spitze verursacht Stoßbelastungen auf Getriebe, Schnecke und Lager. Der Sanftanlauf über Frequenzumrichter (VFD) erhöht die Frequenz von 0 Hz auf 50 Hz innerhalb einer einstellbaren Zeit (5–30 s). Der Anlaufstrom überschreitet nicht 100–150% des Nennstroms, Stoßbelastungen entfallen. Dadurch wird die Belastung des Getriebes deutlich reduziert. Die Lebensdauer des Getriebes steigt um das 2- bis 3-fache.

2. Vektorregelung für Doppelschneckenextruder

Für Extruder optimal ist die sensorlose Vektorregelung (SVC). Der Umrichter modelliert die Rotorlage anhand von Strom und Spannung und liefert ein Anzugsmoment von 150–200% schon bei 1–5 Hz. Dies ist besonders wichtig für die Verarbeitung von Sekundärkunststoffen, die oft ein höheres Anlaufdrehmoment erfordern. Die Vektorregelung eines Asynchronmotors für einen Doppelschneckenextruder ermöglicht ein präzises Drehmomentmanagement auch bei niedrigen Drehzahlen.

2.1 S-förmige Beschleunigungsrampe

Statt linearer Frequenzerhöhung wird eine S-Kurve verwendet: Anfangs niedrigere Beschleunigung (20–50% der mittleren), dann linear, zum Ende wieder reduziert. Das eliminiert Rucke beim Losfahren und Erreichen der Nenndrehzahl.

3. Parametrierung für Sekundärkunststoffe

Startfrequenz (F_start) 1–3 Hz, Hochlaufzeit (Ramp up) 10–30 s (für 75–90 mm Doppelschnecken), Strombegrenzung 120–150% des Motor-Nennstroms. Die Anwendung eines Frequenzumrichters für die Verarbeitung von Sekundärpolymeren ist besonders effektiv, da der bremslose Direktstart sonst zu Reklamationen führen kann.

🔧 Praxisbeispiel: Doppelschneckenextruder 75 mm (90 kW) – Hochlaufzeit 20 s, S‑Kurve 30/30%, Strombegrenzung 200 A (125%). Motor erwärmt sich auf 65°C statt 80–90°C beim Direktstart.

4. Modbus TCP Steuerung unter Linux

Die Anbindung des Frequenzumrichters an ein Linux-basiertes Steuerungssystem (z.B. Linux Mint) erfolgt über Modbus TCP. Damit lässt sich der Sanftanlauf des Extruders vollständig automatisieren. Das folgende Python-Skript realisiert den PID-Regler für die Extruderleistung und steuert gleichzeitig den VFD:

from pyModbusTCP.client import ModbusClient
import time

client = ModbusClient(host='192.168.1.100', port=502)
client.open()

# PID-Regler für Extruderdurchsatz
def pid_controller(setpoint, pv, kp, ki, kd, dt, integral, prev_error):
    error = setpoint - pv
    integral += error * dt
    derivative = (error - prev_error) / dt
    output = kp * error + ki * integral + kd * derivative
    return output, integral, error

# Sanftanlauf
client.write_single_register(8192, 0x0001)  # Start mit 0 Hz
for freq in range(100, 5001, 100):          # 1 bis 50 Hz
    client.write_single_register(8193, freq)
    time.sleep(0.4)                         # 20 s Rampenzeit

5. Einfluss auf den Maschinenpark

Getriebe: Direktstarter-Stöße verursachen Mikrorisse in Zahnflanken. Bei Sanftanlauf erreicht das Getriebe 25–35 Tsd. Betriebsstunden (statt 10–15 Tsd. bei Direktstart). Schnecke/Zylinder: Spalt bleibt 0,2–0,4 mm über 2–3 Jahre erhalten. Lager: Sanftanlauf ermöglicht Lagerlebensdauer von 25–35 Tsd. Stunden.

6. Energieeinsparung und Netzbelastung

Direktstart: 0,07–0,14 kWh pro Start → 500–1000 kWh/Jahr. Sanftanlauf reduziert Verluste um 70–80% und den Spannungseinbruch von 5–10% auf 1–2%. Durch die Reduzierung der Netzrückwirkungen wird die gesamte Betriebssicherheit erhöht.

7. Typische Fehler vermeiden

  • Zu kurze Hochlaufzeit (5–10 s) bei kaltem Extruder → Strom über 200% → Überlastabschaltung. Lösung: beim Kaltstart 30–60 s einstellen.
  • Kein PTC-Widerstand angeschlossen → Überhitzung der Wicklungen (>150°C).
  • Strombegrenzung auf 100% → Start unmöglich bei kaltem Zustand. Auf 140–150% erhöhen (kurzzeitig).

8. Linux‑basierte Automatisierung der gesamten Linie

Die Automatisierung der Extrusionslinie mit Linux Mint umfasst die Überwachung von Temperaturen, Drücken und Drehzahlen. Alle VFDs werden über Modbus TCP zentral gesteuert. Eine benutzerdefinierte PID-Regelung in Python sorgt für eine konstante Schmelzequalität und reduziert den spezifischen Energieverbrauch.

9. Fazit

Der Sanftanlauf über Frequenzumrichter schützt Getriebe, Schnecke und Lager vor mechanischen Überlastungen, senkt den Energieverbrauch und ermöglicht einen zuverlässigen Start auch bei kaltem Extruder. Die Kombination mit einem Linux-Steuerungssystem und Modbus TCP erlaubt eine vollautomatische, reproduzierbare Prozessführung in der Kunststoffrecyclingindustrie.

Soft Start of Extruders with VFD
Linux Modbus TCP Control & Gearbox Protection

Author: Igor Kryazhev, Process Consultant & Polymer Technologist

1. Why Direct Starting Destroys Extruders

Direct starting creates inrush current 5–7× higher than rated. Soft start via VFD ramps frequency from 0 to 50 Hz within adjustable time, inrush current ≤150%, reducing gearbox stress significantly.

2. Vector Control for Twin-Screw Extruders

Sensorless vector control (SVC) provides 150–200% starting torque at 1–5 Hz – essential for processing recycled polymers with higher breakaway torque.

3. Parameterization for Recyclates

F_start 1–3 Hz, ramp up 10–30 s, current limit 120–150% of rated motor current.

✅ Example: 75 mm twin‑screw extruder, 90 kW, ramp 20 s, current limit 200 A → motor temp 65°C (vs 80–90°C).

4. Modbus TCP Control under Linux (Python)

from pyModbusTCP.client import ModbusClient
client = ModbusClient(host='192.168.1.100', port=502)
client.open()
client.write_single_register(8192, 0x0001)
for freq in range(100, 5001, 100):
    client.write_single_register(8193, freq)
    time.sleep(0.4)

5. Component Lifespan

Gearbox life 25–35 khours, screw–barrel clearance remains stable for 2–3 years, bearing life 25–35 khours.

6. Energy Savings

Direct start: 0.07–0.14 kWh/start → 500–1000 kWh/year. Soft start reduces losses by 70–80% and voltage dip from 5–10% to 1–2%.

Плавный пуск экструдера с частотным преобразователем
Управление через Linux Modbus TCP и защита редуктора

Автор: Игорь Кряжев, технолог-консультант

1. Почему прямой пуск разрушает экструдер

Прямой пуск асинхронного двигателя создаёт бросок пускового тока в 5–7 раз выше номинального. Для двигателя 55 кВт пусковой ток – 500–700 А. Плавный пуск через частотный преобразователь наращивает частоту от 0 до 50 Гц за 5–30 секунд, пусковой ток не превышает 100–150% – ресурс редуктора увеличивается в 2–3 раза. Таким образом достигается снижение нагрузки на редуктор при плавном пуске экструдера.

2. Векторное управление для двухшнекового экструдера

Оптимален режим векторного управления без датчика (SVC). Устройство обеспечивает пусковой момент 150–200% уже на частоте 1–5 Гц, что критически важно для переработки вторичных полимеров с высоким начальным сопротивлением. Внедрение векторного управления асинхронным двигателем для двухшнекового экструдера позволяет точно дозировать крутящий момент даже на малых оборотах.

⚙ Пример: двухшнековый экструдер 75 мм (двигатель 90 кВт, номинальный ток 160 А) – время разгона 20 с, S-кривая 30/30%, ограничение тока 200 А (125%). Температура двигателя после пуска 65°C (при прямом пуске 80–90°C).

3. Управление через Modbus TCP из Linux

Частотный преобразователь подключается к промышленному компьютеру под управлением Linux Mint через Modbus TCP (порт 502). Это позволяет реализовать плавный пуск экструдера с частотным преобразователем через Linux Modbus TCP без дополнительных контроллеров. Пример на Python с ПИД-регулированием производительности:

from pyModbusTCP.client import ModbusClient
import time

client = ModbusClient(host='192.168.1.100', port=502)
client.open()

# ПИД-регулятор производительности экструдера
def pid_controller(setpoint, pv, kp, ki, kd, dt, integral, prev_error):
    error = setpoint - pv
    integral += error * dt
    derivative = (error - prev_error) / dt
    return kp * error + ki * integral + kd * derivative, integral, error

# Плавный пуск
client.write_single_register(8192, 0x0001)
for freq in range(100, 5001, 100):
    client.write_single_register(8193, freq)
    time.sleep(0.4)

Такая схема позволяет полностью автоматизировать экструзионную линию на базе Linux Mint, интегрируя управление скоростью, температурой и давлением в единую программу.

4. Влияние на ресурс оборудования

Редуктор: 25–35 тыс. часов (вместо 10–15 тыс.). Шнек и цилиндр: зазор сохраняется 0,2–0,4 мм в течение 2–3 лет. Подшипники: 25–35 тыс. часов. Применение частотного преобразователя для переработки вторичных полимеров оправдано не только экономией энергии, но и продлением межремонтного интервала.

5. Энергопотребление и защита сети

Прямой пуск: потери 0,07–0,14 кВт·ч на пуск → 500–1000 кВт·ч/год (60–120 €). Плавный пуск снижает потери на 70–80% и просадку напряжения с 5–10% до 1–2%.

6. Частотный преобразователь для вторичных полимеров – критерии выбора

  • Мощность: не менее мощности двигателя
  • Номинальный ток: на 15–20% выше тока двигателя
  • Перегрузочная способность: 150% в течение 60 секунд
  • Режимы: векторный без датчика (SVC) и V/f
  • Интерфейсы: Modbus TCP для управления из Linux

7. Типичные ошибки и их предотвращение

  • Слишком короткое время разгона (5–10 с) для холодного экструдера → увеличить до 30–60 с.
  • Отключён датчик температуры двигателя (PTC-термистор) → перегрев обмоток.
  • Ограничение тока 100% от номинала → невозможность пуска → установить 140–150% кратковременно.

8. Заключение

Комбинация частотного преобразователя с векторным управлением и Linux‑системы на базе Modbus TCP даёт максимальный эффект для recycling-экструдеров. Снижаются пиковые нагрузки, защищается редуктор, упрощается автоматизация. Настройка ПИД-регулятора производительности экструдера на Python позволяет поддерживать стабильный выход продукта при изменении свойств вторичного сырья.
